## Version 3.0.0 (current)

Version 3.0.0 is the current development branch. The project and library name changed from `libsrtp2` / `srtp2` to `libsrtp3` / `srtp3`, and the CMake target is now `libSRTP::srtp3`.

<Warning>
Version 3.0.0 contains breaking API changes relative to 2.x. The SSRC parameter to `srtp_stream_remove()` changed from network byte order to host byte order, consistent with the rest of the API.
</Warning>

**Breaking changes**

`srtp_stream_remove()` now takes the SSRC in host byte order. In libSRTP 2.x and earlier, the parameter was in network byte order.

The CMake library target is `srtp3` (alias `libSRTP::srtp3`). Projects that linked against `srtp2` must update their CMake target name.

## Version 2.1.0

<Warning>
Version 2.1.0 contains compatibility breaks for RTCP streams using AES-256-ICM with OpenSSL, RTCP streams using AES-GCM (little-endian), and header extension encryption with AES-GCM. These were bug fixes for incorrect behavior.
</Warning>

<AccordionGroup>
<Accordion title="Breaking changes (compatibility)">

**PR #253** — Fixed an incorrect IV formation for AES-256 ICM when used with RTCP and OpenSSL. Breaks compatibility with libSRTP 1.5 and 2.0 for RTCP streams using AES-256-ICM with OpenSSL.

**PR #259** — Fixed the AES-GCM IV construction on little-endian machines. Breaks compatibility with libSRTP 1.5 and 2.0 for RTCP streams using AES-GCM.

**PR #287** — Fixed an out-of-bounds read in key generation for encrypted header extensions with GCM ciphers by padding the GCM salt. Breaks compatibility with libSRTP 2.0 when using header extension encryption with AES-GCM.

</Accordion>
<Accordion title="New features">

**PR #224** — Added MKI (Master Key Identifier) support with up to 4 keys per session.

**PR #289** — Added `srtp_stream_set_roc()` and `srtp_stream_get_roc()` to get and set the Roll-Over Counter (ROC).

**PR #241, #261** — Improved logging API: ability to receive log messages from libSRTP, enable logging to stdout or a file, and control individual debug modules.

**PR #234** — Changed `srtp_event_data_t` to report the SSRC instead of the opaque `srtp_stream_t` pointer, making the event callback framework useful in practice.

**PR #204** — OpenSSL performance improvements: key expansion now occurs once per key instead of once per packet.

**PR #209** — Restored AES-192 support under BoringSSL.

</Accordion>
<Accordion title="Bug fixes">

**PR #304** — Fixed SRTP and SRTCP packet processing on big-endian machines.

**PR #192** — Replaced `octet_string_is_eq` with a constant-time implementation to prevent timing side-channels.

**PR #294** — Fixed incorrect result of `rdb_increment` on overflow.

**PR #182** — Added a length check before reading packet data.

</Accordion>
</AccordionGroup>

## Earlier versions

Versions prior to 2.1.0 are not tracked in the `CHANGES` file in the same structured way. For historical releases and their tags, see the [GitHub releases page](https://github.com/cisco/libsrtp/releases).

Key milestones:

**2.0.0** — Major rewrite introducing the `srtp_` prefix on all public symbols, breaking compatibility with the 1.x API.

**1.6.0** — Final 1.x release before the 2.0 rewrite.

**1.5.x** — Long-running stable series used by WebRTC implementations.
